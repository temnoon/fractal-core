fractal-core is the canonical pulse server for the humanizer family of web services — humanizer.com, post-social.com, gravity-press.com, temnoon.com — and any federated operator that adopts the protocol. Every act of communication can be stamped with a signed pulse: a cosmic clock value T bound to a freshly-minted prime P.

The math is open. The signing is Ed25519. The interface uses no AI agent at runtime.

What a pulse is

Pick an anchor (a moment in time you all agree existed). Pick a tick unit (Planck time, an Earth day, a Yad-second, a millisecond). The clock value T is the integer number of ticks since the anchor. T is monotonic, deterministic, and identical for every observer — that's all a clock needs to be.

The minted prime P is a separate value. Each mint draws a uniform random number from a 2120-wide window in a chosen bit-class, then climbs by 2 until Baillie–PSW confirms a prime. Two mints during the same tick produce different P with overwhelming probability, so P uniquely identifies the act of minting; T orders mints in time; Ed25519 binds them together.

Four canonical systems

cosmic

Anchor: 16×10⁹ Julian years before the Carrington Event (1859-09-01T11:18:00Z).
Tick: Planck time (5.391247×10⁻⁴⁴ s).
Today's T ≈ 2²⁰³. Prime defaults to 256 bits.

tonga

Anchor: ~74,000 years before J2000 (Toba supervolcano).
Tick: one Earth day (86,400 s).
One prime per day, cached. Today's T ≈ 2.7×10⁷.

yad

Anchor: J2000 epoch.
Tick: one Yad-second (sidereal year / 512 / 61637 ≈ 1.000016 s).
One prime per Yad-second, cached. Yad = day backwards.

milli

Anchor: 2026-01-01T00:00:00Z.
Tick: one Earth millisecond.
Fresh prime per mint. Today's T ≈ 10¹⁰.

Endpoints

GET /api/v1/pulse → list systems + signing keys GET /api/v1/pulse/{system_id}/parameters → describe a system, federation-ready GET /api/v1/pulse/{system_id}/now → public, signed pulse only POST /api/v1/pulse/{system_id}/mint → auth required, target_domain bound POST /api/v1/pulse/{system_id}/verify → verify a signed pulse

Mint a pulse

curl -X POST https://api.fractal-core.com/api/v1/pulse/cosmic/mint \
  -H "Authorization: Bearer frc_..." \
  -H "Content-Type: application/json" \
  -d '{
    "target_domain": "post-social.com",
    "nonce": "0123456789abcdef0123456789abcdef",
    "purpose": "channel-anchor",
    "verbose": true
  }'

Verify a pulse

curl -X POST https://api.fractal-core.com/api/v1/pulse/cosmic/verify \
  -H "Content-Type: application/json" \
  -d '{ "signed": <signed pulse JSON> }'

Federation

The protocol is open. To run your own time system, publish your time_system descriptor (anchor, tick unit, age offset, mint cadence, window bits, signing key) at your own /parameters endpoint, and sign your pulses with Ed25519. Verifiers don't care which operator signed — they need only your public key and the canonical bytes.

The four systems above are fractal-core's instances of the protocol. They aren't the protocol.

Disclosure. Code authored with the assistance of language models. The API itself and this interface use no AI agent at runtime — only deterministic prime mathematics and Ed25519 signing.
